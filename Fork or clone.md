۶. زیرِ هود (وقتی Fork یا Clone می‌کنی)

موقع Fork:
۱.GitHub یک کپی کامل از همه فایل‌ها و تاریخچه برات می‌سازه
۲.مالکیت جدید: این کپی مال توئه و می‌تونی هر کاری روش انجام بدی
۳.ارتباط با پروژه اصلی حفظ می‌شه تا بعداً بتونی Pull Request بدی

موقع Clone:
۱.همه فایل‌ها و تاریخچه Git روی کامپیوترت دانلود می‌شه
۲.یک پوشه محلی با همه کدها ایجاد می‌شه
۳.Git به طور خودکار آدرس پروژه اصلی رو ذخیره می‌کنه (به نام origin)
